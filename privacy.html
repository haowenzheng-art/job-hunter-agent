隐私政策

最近更新：2026 年 6 月 26 日

JobHunter（以下简称"我们"）重视你的隐私。本政策说明我们收集什么数据、如何使用、如何保护，以及你享有哪些权利。使用本服务即视为你同意本政策。

1. 我们收集的数据

  • 账号信息：注册时提交的用户名、密码（PBKDF2 加盐哈希存储，明文不可逆）。
  • 简历数据：你在 Flow A / Flow B 中输入或生成的基础信息、工作经历、项目经历、技能、个人陈述等。
  • JD 库数据：公开招聘平台公开 JD（已脱敏）+ 用户手动添加的 JD。
  • 使用日志：访问时间、页面路径、操作行为，用于排查问题和优化体验。

2. 数据用途

  • 生成定制简历、RAG 检索召回、差距分析。
  • 调用 LLM 时仅传递改写所需的最小上下文，不留存在第三方。
  • 统计聚合（如岗位热度、技能分布），结果不关联到具体用户。
  • 不用于任何商业广告投放，不出售给第三方。

3. 数据存储

  • 默认存储在本地 SQLite 数据库（data/jobhunter_v2.db），文件级隔离。
  • 进阶部署可使用 PostgreSQL + pgvector，数据存放在你自建的服务器。
  • 不使用云端托管数据库，你的简历数据不出你的基础设施。
  • 密码使用 PBKDF2 加盐哈希，原始密码不入库。

4. LLM 调用

  • LLM 调用通过你配置的 API Key 直接发起，请求不经过我们的服务器中转。
  • 仅传递简历改写所需的最小上下文片段，不传递完整简历全文。
  • LLM 服务商（如 Agnes、火山引擎、OpenAI、DeepSeek）的处理受其各自隐私政策约束。

5. 第三方服务

  • 招聘平台（JobsDB、猎聘、Boss、51job 等）：仅采集公开 JD 数据，不涉及登录凭证。
  • LLM 提供商：按其隐私政策处理请求内容。
  • 不接入 Google Analytics、Facebook Pixel 等追踪广告 SDK。

6. Cookie 使用

本服务使用 Streamlit 内置 session cookie 维持登录状态，不放置追踪 Cookie，不进行跨站行为分析。

7. 数据保留与删除

  • 账号和简历数据保留至你主动删除为止。
  • JD 库中的公共爬取数据采用软删除（标记 deleted_at），不会物理删除以保留历史关联。
  • 你可在"账号设置"中导出或删除你的全部数据。

8. 你的权利

  • 访问权：随时查看你提交的所有数据。
  • 更正权：随时修改简历信息和账号资料。
  • 删除权：随时删除账号及关联数据。
  • 导出权：以 Markdown / JSON 格式导出你的简历数据。

9. 未成年人保护

本服务面向 18 岁及以上求职者。如发现未成年人注册，我们将立即删除其账号和数据。

10. 政策更新

本政策可能随服务演进更新。重大变更将在首页和登录页提示，继续使用即视为接受更新后的政策。

联系我们

如对本政策有疑问，请发送邮件至 privacy@jobhunter.dev，我们将在 3 个工作日内回复。
